Operating Rule

Hub 只做路由器，不和专站抢内容。

正确的分工不是把所有设计内容揉在一起，而是把不同维度拆成独立站点，再由总入口解释它们的关系。

01
Hub 负责分流 告诉用户现在该去哪一站，而不是重复建设第二套 atlas。
02
专站负责深内容 截图、时间轴、prompt、skills、案例和方法论都应该沉到对应维度。
03
仓库负责可复用资产 `design-skills` 是共享层，用来沉淀可以被 AI 重复调用的约束包。

Boundaries

每一站负责什么，不负责什么。

这是整个设计产品线最重要的纪律。只有边界明确，后面的 `webdesign / ux / icon / skills` 才不会再次合并成一个混乱的大站。

01 · Webdesign

网页风格 atlas

负责

风格家族、真实参考网站、场景匹配、Prompt / Skill 下载、网页设计语言的浏览与比较。

不负责

完整 UX 审计、图标识别系统、所有 design 子站的总导航。

输出：风格方向、参考链接、可复用约束。

02 · UX

体验与清晰度

负责

用户路径、信息可理解性、界面清晰度、服务断点与产品体验层面的判断。

不负责

代替网页风格 atlas 做风格图库，也不负责图标识别细节。

输出：问题定位、改造建议、服务与交互层修正。

03 · Icon

图标与识别系统

负责

图标风格、app icon、favicon、黑白可识别度、尺寸下的辨认与系统一致性。

不负责

整站布局、阅读节奏、历史流派归纳与大篇幅页面视觉体系。

输出：图标方向、识别规则、可用尺寸资产。

04 · Shared Layer

Design Skills

负责

把各站抽象出的 Prompt、Skill 文件和 AI 可复用规则沉到统一仓库。

不负责

直接承担用户浏览体验，也不应该取代人类可读的站内解释层。

输出：可下载、可复用、可同步的 AI 设计约束。
